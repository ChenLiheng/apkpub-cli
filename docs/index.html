一条命令
分发到所有应用市场

apkpub-cli 是一个面向开发者的 APK 多市场分发工具。
一条命令将已签名 APK 发布到华为、小米、OPPO、VIVO、荣耀及自定义渠道。

npm install apkpub-cli 快速开始
terminal 
# 一键发布到华为 + 小米
$ apkpub publish --app com.example.app --apk ./app-release.apk \
  --channels huawei,mi --desc "版本 2.1.0 更新" --yes

痛点

APK 分发很痛苦

  • 每个应用市场有各自的上传 API 和签名算法
  • 手动上传耗时且易出错，重复劳动
  • CI/CD 流程中难以集成多渠道发布
  • 密钥和凭证散落在各处，缺少统一管理
  • 审核状态需要逐个平台登录查看
  • AI Agent 无法直接操作应用市场
方案

一条命令搞定一切

  • 统一 CLI 接口屏蔽各平台 API 差异
  • 多渠道并行上传，支持预检模式
  • 原生 CI/CD 集成，管道安全可解析
  • 分层密钥管理：环境变量 → keychain → 加密文件
  • 市场状态一键查询，审核动态一目了然
  • 内置 MCP Server，AI Agent 无缝调用

快速开始

30 秒上手

安装、配置、发布，三步走完。

安装 #1 
$ npm i -g apkpub-cli
# 或
$ pnpm i -g apkpub-cli
初始化配置 #2 
$ apkpub init
# 交互式向导，自动创建
# ~/.apkpub/apps/{appId}.json

$ apkpub init \
    --name "我的应用" \
    --app com.example.app \
    --channels huawei,mi
查询市场状态 #3 
$ apkpub status --app com.example.app
华为  ✔ 已上线  v215
小米  ✔ 已上线  v210
OPPO  ◐ 审核中  v215

$ apkpub status \
    --app com.example.app --json
发布 APK #4 
# 单包发布
$ apkpub publish --app com.example.app \
    --apk ./app-release.apk \
    --channels huawei,mi,oppo \
    --desc "版本更新" --yes

# 预检模式（不上传）
$ apkpub publish --app com.example.app \
    --apk ./app.apk --dry-run --json
架构设计

分层 · 扩展 · 安全

bin/apkpub.ts cli/init cli/publish cli/status cli/doctor cli/config mcp/server cli/describe
↓ CLI 编排
Dispatcher TaskLauncher versionCheck result
↓ Channel 接口契约
huawei honor mi oppo vivo oss http
↓ 安全隔离
assertSafeUrl sanitizePath redactMessage

清晰的分层架构 — CLI 薄层只做参数解析与输出，核心编排层通过 Channel 接口驱动渠道实现，新增渠道只需实现统一契约。


publish 核心数据流：

loadConfig(appId)
→ resolveChannels
→ Dispatcher.dispatch()
  └─ TaskLauncher.prepare()
     ├─ resolveConfigSecrets
     ├─ selectFile + parseApk
     ├─ checkPackageMatch
     ├─ getMarketState + checkVersion
     └─ channel.upload()
→ aggregateResults → exitCode


渠道隔离 — 单渠道失败不影响其他渠道，每个渠道独立 try/catch。最终退出码精确反映结果（0=全成功 / 4=部分失败 / 5=全失败）。

CI/CD

GitHub Actions 集成

非交互模式 + --json + --no-progress，管道安全。退出码精确反映发布结果。

- name: 发布 APK env: HUAWEI_CLIENT_SECRET: ${{ secrets.HUAWEI_CLIENT_SECRET }} MI_PRIVATE_KEY: ${{ secrets.MI_PRIVATE_KEY }} run: | apkpub publish \ --app com.example.app \ --apk ./build/outputs/apk/release/ \ --channels huawei,mi \ --desc "CI 自动发布" \ --yes --json --no-progress
AI Agent

MCP Server 集成

以 stdio 启动 MCP Server，暴露 4 个工具供 AI Agent 调用。Cursor、Claude 等工具开箱即用。

// Cursor MCP 配置 { "mcpServers": { "apkpub": { "command": "apkpub", "args": ["mcp"] } } }
apkpub_info apkpub_status apkpub_publish apkpub_doctor
契约

稳定退出码

发布结果通过退出码精确表达，CI/CD 和 Agent 可安全解析。

退出码 含义 说明
0 全部成功 所有渠道上传完成
1 内部错误 非预期的运行时异常
2 参数/配置错误 CLI 参数无效或配置文件损坏
3 版本校验失败 全部渠道因 versionCode 过低失败
4 部分渠道失败 至少一个渠道上传成功，至少一个失败
5 全部失败 所有渠道均上传失败
安全

安全内置，无妥协

处理市场凭证等敏感信息，apkpub 在每一层都做了防护。

🔒 配置权限
~/.apkpub/ 目录 0700，配置文件 0600
🔑 密钥加密
AES-256-GCM + scrypt，环境变量优先
👁️ 日志脱敏
secret/token/privateKey 自动 ***REDACTED***
🛡️ SSRF 防护
禁止内网地址，仅允许 http/https
🔗 路径穿越防护
禁止 .. 与绝对路径
📤 输出分流
JSON→stdout，日志→stderr，管道安全
