一条命令
分发到所有应用市场

apkpub-cli 是一个面向 Android 开发者的多市场分发工具。一条命令将已签名 APK 发布到华为、小米、OPPO、VIVO、荣耀及自定义渠道。支持 CI/CD 和 AI Agent 集成。

npm install apkpub-cli 快速开始

痛点

APK 分发很痛苦

  • 每个应用市场有各自独立的上传 API 和签名算法
  • 手动上传耗时且易出错，重复劳动
  • CI/CD 流程中难以集成多渠道发布
  • 密钥和凭证散落在各处，缺少统一管理
  • 审核状态需要逐个平台登录查看
  • AI Agent 无法直接操作应用市场
方案

一条命令搞定

  • 统一 CLI 接口屏蔽各平台 API 差异
  • 多渠道并行上传，支持预检模式
  • 原生 CI/CD 集成，管道安全可解析
  • 分层密钥管理：环境变量 → keychain → 加密文件
  • 市场状态一键查询，审核动态一目了然
  • 内置 MCP Server，AI Agent 无缝调用
功能特性

为工程化而设计

同时面向本地开发者、CI/CD 流水线和 AI Agent 三种使用场景。

★

多市场分发

同时发布到华为、小米、OPPO、VIVO、荣耀等应用市场。并行上传、进度可见、逐一报错互不影响。支持多渠道包按文件名标识自动匹配。

apkpub publish --app com.example.app --apk ./apks/ --channels huawei,mi,oppo,vivo,honor

🔒

分层密钥管理

环境变量 → 系统 Keychain → AES-256-GCM 加密 → 明文。自动脱敏日志与输出。

🤖

MCP / Agent 集成

内置 MCP Server，暴露 info / status / publish / doctor 工具。Cursor、Claude 等 Agent 可直接调用。

⚡

CI/CD 原生集成

非交互模式、--json 机器可读输出、稳定退出码契约。GitHub Actions 一键接入。

🔄

自定义上传渠道

阿里云 OSS（AK/SK 或 STS）+ 通用 HTTP 上传。模板化路径，灵活适配私有分发链路。

🛡

安全默认值

SSRF 防护、路径穿越防护、配置目录 0700、日志脱敏、输出分流（JSON→stdout，日志→stderr）。

快速开始

30 秒上手

安装、配置、发布，三步走完。

安装1
$ npm i -g apkpub-cli
# 或
$ pnpm i -g apkpub-cli
初始化配置2
$ apkpub init
# 交互式向导

$ apkpub init \
  --app com.example.app \
  --channels huawei,mi
查询状态3
$ apkpub status \
  --app com.example.app
华为  ✓ 已上线  v215
小米  ✓ 已上线  v210
OPPO  ◊ 审核中  v215
发布 APK4
$ apkpub publish \
  --app com.example.app \
  --apk ./app-release.apk \
  --channels huawei,mi,oppo \
  --desc "版本更新" --yes
集成

CI/CD 和 AI Agent 双通道

无论你习惯 YAML 流水线还是自然语言指令，apkpub 都已准备就绪。

CI/CD

GitHub Actions

非交互模式 + --json + --no-progress，管道安全。退出码精确反映发布结果。

- name: 发布 APK
env:
HUAWEI_CLIENT_SECRET: ${{ secrets.HUAWEI }}
run: |
apkpub publish --app com.example.app \
  --apk ./build/outputs/apk/release/ \
  --channels huawei,mi \
  --desc "CI 自动发布" --yes --json --no-progress
AI Agent

MCP Server

以 stdio 启动 MCP Server，暴露 4 个工具供 AI Agent 调用。Cursor、Claude 开箱即用。

// Cursor MCP 配置
{
  "mcpServers": {
    "apkpub": {
      "command": "apkpub",
      "args": ["mcp"]
    }
  }
}
apkpub_info apkpub_status apkpub_publish apkpub_doctor
安全

安全内置，无需担忧

处理市场凭证等敏感信息，apkpub 在每一层都做了防护。

🔒配置权限

👁日志脱敏
secret/token 自动 ***REDACTED***
🛡SSRF 防护
禁止内网地址，仅 http/https
🔗路径穿越防护
禁止 .. 与绝对路径
📤输出分流
JSON→stdout, 日志→stderr
