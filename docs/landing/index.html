Why apkpub

将逐平台手动上传，
沉淀为一条稳定的发布流水线。

一份 APK，多渠道并行分发。

读取配置~/.apkpub
解析 APKpackage/version
校验线上版本market state
上传与审计result + log
01

屏蔽各市场 API 差异

上传签名、鉴权、版本查询和错误归集都由渠道实现处理，核心编排只关心 Channel 契约。

02

单渠道失败不影响整体发布

每个渠道独立 try/catch，单渠道失败会进入结果汇总，其他渠道继续推进。

03

输出同时面向人与机器

交互式向导服务本地使用，--json 与稳定退出码服务 CI/CD，MCP 服务 Agent。

Capabilities

为真实发布场景设计，
而非演示样例。

apkpub-cli 的设计重点是凭证安全、版本校验、可取消上传、可审计结果，以及能被自动化系统稳定调用。

publish orchestration

多渠道并发编排

目录模式按渠道标识自动选包，单文件模式复用同一 APK；进度、失败和审计日志按渠道独立记录。

agent-native

自描述 CLI + MCP Server

apkpub describe --json 暴露能力清单，apkpub mcp 暴露 info/status/publish/doctor 工具。

secrets

分层密钥解析

环境变量、系统 Keychain、AES-256-GCM 加密配置和明文回退，日志默认脱敏。

ci contract

稳定退出码

参数错误、版本过低、部分失败和全部失败都有明确退出码，方便流水线分支处理。

security

默认安全

外发 URL 经过 SSRF 防护，模板路径禁止穿越，JSON 输出与日志输出分流。

主流市场开箱即用，私有渠道灵活扩展。

内置市场渠道走统一 Channel 接口；私有分发可以通过 OSS 或通用 HTTP 上传接入。

Automation

CI/CD 与 Agent 共用同一套核心链路。

CLI 命令与 MCP 工具复用 core 编排，避免本地、流水线、Agent 三套行为分叉。

GitHub Actions

面向流水线的可解析结果

--json 只输出机器结果到 stdout，进度和日志走 stderr，方便管道解析。

apkpub publish --app com.example.app \
  --apk ./build/outputs/apk/release/ \
  --channels huawei,mi \
  --desc "CI 自动发布" \
  --yes --json --no-progress

MCP Server

让 Agent 既能发现，也能执行

使用 stdio 启动 MCP Server，让 Agent 查询信息、状态、预检并触发发布。

{
  "mcpServers": {
    "apkpub": {
      "command": "apkpub",
      "args": ["mcp"]
    }
  }
}
apkpub_info apkpub_status apkpub_publish apkpub_doctor

Security

发布工具，更要守住安全边界。

apkpub-cli 对凭证、URL、路径和日志输出都做了明确约束，降低自动化发布的操作风险。

配置权限~/.apkpub 目录 0700，配置文件 0600。
密钥加密AES-256-GCM + scrypt，支持环境变量和 Keychain。
日志脱敏secret、token、privateKey 等敏感字段自动隐藏。
SSRF 防护上传和回调 URL 限制协议并拦截内网地址。
路径防护模板路径禁止 .. 与绝对路径。
输出分流JSON 走 stdout，日志和进度走 stderr。
